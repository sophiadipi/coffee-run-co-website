minimum requirement features:

html5:

  • Appropriate HTML5 tags → see source code.
  • Inline images; both static and clickable → clickable images in the highlight boxes on the home page and static images on the about, events, and merchandise pages.
  • Use of appropriate special characters → copyright symbol (©) in footer and other special characters used throughout the website.
  • Use of appropriate <meta> tags → see source code. <meta> tags used in each html document to specify character set and to control the viewport width and initial scale.
  • A footer on the home page indicating when the website was last updated or modified → see home page footer.

css3:

  • Use CSS3 to format the appearance of paragraphs, lists, headers, background images, tables, and forms → see stylesheets.
  • Use CSS3 contextual selectors and at least two attribute selectors, excluding class and id attribute selectors → see stylesheets.
  • Use CSS3 styles for colour (RGBA & HSLA) → see stylesheets. RGB(A) colour values used throughout.
  • Use dynamic pseudo-classes to highlight hyperlinks states of link (visited, active, and hover), focus, and two structural pseudo-classes (first-of-type and last-of-type) → see stylesheets.
  • Use CSS3 background styles → used for the hero image on the home page.See home.css.
  • Your home page should have a drop cap created with CSS3 → see the introductory paragraph on the home page.
  • Define document content with the class and id properties, and create styles for them → see html documents and corresponding stylesheets.
  • Use CSS3 text and box shadows → examples throughout site.
  • Use CSS3 transform rotate feature → see web form.
  • Use CSS3 opacity applied to two images to appear semi-transparent → see about.css and events.css

web form:

  • Use appropriate form elements including some of the new HTML5 form elements → see the web form and the join.html source code file.
  • Apply appropriate styles to web forms → see the web form and join.css
  • Apply inline form validation by using the focus pseudo-class to appropriate form elements → see join.css

multimedia:

  • Use sound and video somewhere on your website → see the video on the about page.
  • Use transitions and animations → animation example in hero section of the home page. See home.css. Transitions were used for hover styles for most of the hyperlinks on the website to create a gradual colour changing effect rather than a harsh and abrupt change. Hover over any hyperlink on this page for a demonstration. See base.css.

javascript:

  • Use variables; arithmetic, comparison and logical operators; subroutines; functions; arrays; conditional statements; conditional operators; and running time commands with system clock → see JavaScript files (upcomingEvents.js, merch.js, and formSubmit.js).
  • Use system date on your website → I used system date to create the event countdown timer. See upcomingEvents.js.
  • Validate form input by using appropriate JavaScript functions → see formSubmit.js
  • Use JavaScript alerts for validation purposes → see formSubmit.js

responsive design:

  • Your website design should include responsive design implementation using concepts covered in this course that would accommodate desktop, tablet, and mobile designs, including positioning, grid, and flex layouts → see stylesheets. Media queries were used to make each page responsive.
  • The use of contextual selectors to control sub-level menus is required, including a navicon image for mobile design → see the <header> section of any of the html documents and base.css.
  • Images should also be responsive for different screen sizes and devices → see stylesheets.
